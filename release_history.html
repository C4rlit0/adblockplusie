Background

Windows Installer uses a few pieces of information to manage its upgrade process.

  • Product ID. This is a GUID that names a particular product release; it's the primary name. It has to be changed with each release. Within WiX, this is the attribute Product/@Id. Within the Windows Installer, this is the ProductCode property. Caution: The property called ProductID in Windows Installer is called Product/@Name in WiX.
  • Version. This is an informational field, which is dependent upon the Product ID. The version number is a dotted triple, with an optional fourth field. For comparison and equality, the first three fields are used and the fourth is ignored. Within WiX, this is the attribute Product/@Version. Within the Windows Installer, this is the ProductVersion property.
  • Upgrade Code. This GUID designates membership in a category that all may be upgraded by another, common package. Colloquially, it names what we would call the "Product", irrespective of revision. Technically, it's an identifier that allows one package to declare that it is able to supersede another package. Within WiX, this is the attribute Product/@UpgradeCode. Within the Windows Installer, this is the UpgradeCode property.
  • Package ID. This is a GUID that identifies a particular .msi file (in WI lingo, a particular package). Within WiX, this is the attribute Product/Package/@Id. Within the Windows Installer, this is called a package code. We never need to care about this, because the WiX syntax Id="*" generates unique identifiers each time. It's documented here in order to inform the reader that they may ignore it without peril.
